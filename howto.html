1 Getting started


1.1 Ways to start the
application


1.1.1
Executing the jar file







Copy the jar-file and the lib-folder to
a folder the user has write access to


E.g. /home/<username>/jprinteradmin
or c:\users\<username>\jprinteradmin.


Make a shortcut e.g. a
jprinteradmin.bat file with the content “javaw -jar
<pathToJarFile>”.







1.1.2
Using Web-Start


Place the whole folder to a web-server.


Add the certificate to signer-ca in the
java control panel under “security” → “certificates”


Navigate your Browser to the place
where you put the folder in.


Start the launch.jnlp file.


Trust the application, if you wants to
start it


You can make short-cut from the java
control panel by navigate to “Basic” → “View temporary files”
→ Right Click on jprinteradmin → install shortcut







Tip: If you use this location to start
the server-application (as described in point 2.5) the server and the
clients always uses the same program-version.







1.2
Choose your database


if you don't need access for more then
one user or want to let a server-job let run jobs (e.g. Reading the
printers data periodically) the local database (sqlite) would be the
best choice. 



But if you need a server or several
users you must choose between Microsofts SQL-Server and mysql.



1.3 Creating a database
and connect to it



in the menu-bar click "Settings" and then on the entry
"Connection". The Connection-Window will appears. Click on
the button "create database" then in the combo-box
“database” choose your database. In case of a local database all
you need to do is to click the button "create".
jprinteradmin will create a sqlite database-file in your appdata
folder in the sub-folder ".jprinteradmin". If you choose
one of the others you have to fill in host, port ( the default will
be refilled ), administrator credentials, the name of the database
that will be created and the credentials of the user that will be
created to connect to the database.


Click create. In the Connection-Window
click "Connect".







tip: if something went wrong you have
to delete the database-file in case of a local installation or delete
the database and the created user in case of an MS-SQL-Server or a
MySQL-Server installation.







1.4
Setting up your network-environment







Click "Settings" in the
menu-bar and then "Network".


Fill in first and last IP-address and
give it a name. This can be shown as a field and may be useful if you
have several IP-areas in different locations.


Click on the button "add".


Example:


first address: 192.168.2.200


last address: 192.168.2.250


customer: default network



1.5 Test your
configuration







In the main-window click "read
printer-data". The program should check any IP for a printer and
when ready show them in the main-grid. By double-clicking an entry
you can open then printer-window with all the informations available
for this printer.

2.1 Configure the values
that should be read from the printers







Go to "Settings" ->
"Request-Values-Settings"







For the default-request,method just add
a field by typing in name and fill in the other values:


- view in main window → weather this
value is visible in main window or not


- order where is it visible (if it is)


- search field → if the content of
this value will be included in search from the main-window


- Identification-field → if this
value will be take to identify the printer


- the default
id-field is the mac-address also IP would be possible


- counter → if a field is seen as a
counter every single request-value will be stored in the database.
Also it is possible to calculate sum and average values for these
fields.







Then click add. Now the field is
present, but the program still doesn't know how to retrieve this
value. Therefore you have to choose a request-kind and configure it.


Choose your request-kind in the
combo-box:

one


Click on the button “Change
Request-Settings”. The “SNMP-Settings” Window will appear.
There you can type in an oid e.g. 1.3.6.1.2.1.1.5.0 for the printers
name. You can put in an IP-address from a printer and see the result
this would bring, then you can click “OK”







2. html- this I a bit more complicated
and will be describes in the next version of this document.

4. userdefined – this value can be
modified manually by the user in the main-window or the
printer-window







If you need another Request-Method
(mostly because you have different kind of printers where it is not

2.2
Defining printer-types


Click “Settings” in
the menu-bar and then “Printer-types”


Here you can see all printer-types already found by the
application. You can change the Request-Method by clicking on one and
choose one ( defining of them is described in Point 2.1 of this
document) and click on “update”. You also can type in a new
printer-type. But normally that doesn't make sense, because you don't
know the exact printer-type-name from snmp before you have tried it.








2.3
Defining the general settings


Click “Settings” in the menu-bar and then “General settings”


Here you can define setting that are valid for every user or
server process 



On the left side you have the settings for requesting printers and
on the right side there are the settings for email. Email is only
important if you want to send a report by email.


Note that the smtp-server has to be typed in with port.
E.g. 192.168.2.1:25 for an smtp-server without secure access,

2.5
Setting up a server task


This example is for a linux-server., but it shouldn't be a problem
for any windows-administrator to adapt this procedure to a “Windows
Scheduled Task”


Make sure java (Version 7 or higher) is installed


create a user that run the job :


e.g. “sudo useradd jprinteradmin”


copy your “appdata – .jprinteradmin” to the users
appdata-folder on the server e.g. to
/home/jprinteradmin/.jprinteradmin


Make a cronjob that runs once a day by editing the file
“/etc/crontab”


E.g. Add the line 



“35  7    * * *   jprinteradmin java -jar
/var/www/jPrinterAdmin/jPrinterAdmin.jar planer”








In this case the job is started every day at 07:35am by the user
“jprinteradmin.”


It will look if there is something to do and do it.


To see how to define when there is something to do look at the
next point (2.6) of this document

3.1
Switching to another database


This may be useful if you had decided to use a local database, but
now you want to create a server job. Therefore you need a
database-server that is always present and can handle several users.


To exchange the database you have to make an export by navigate to
“File” → “Export”, choose the folder and type in a name
then click “execute”.


After that you can create and log on to a new database as
described in Point 1.3 of this document.


At the end you can import your backup-file by clicking “File”
→ “Export”, choose your file and click “Open”.


3.2
Deleting a lock


Go to “Extras” → “view locks”


Here you can clear all locks or mark one and delete this one.
