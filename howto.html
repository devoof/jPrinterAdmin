Diese Projekt und dieses Dokument ist ein Teil davon, ist ein reines Hobby-Projekt. Es war nie für eine professionelle Benutzung gedacht. Ich bin für jede Verbesserung der Programmierung oder der Texte dankbar.

1 Start ins Programm


1.1 Wege die Anwendung zu
starten


1.1.1
Ausführen der „jar“-Datei


Kopiere die „jar“-Datei
und den 'lib“-Ordner an eine Stelle auf die der Benutzer
Lese-Rechte hat,


z.B.
/home/<Benutzername>/jprinteradmin oder
c:\users\<Benutzername>\jprinteradmin und erstelle eine
Verknüpfung.


Zum Beispiel eine Datei
namens „ jprinteradmin.bat“ mit dem Inhalt .


“javaw -jar
<pfadZurJarDatei>”








1.1.2
Benutzen von Web-Start


Kopieren den ganzen Ordner
mit jar- und jnlp-Datei usw. samt dem lib Ordner auf einen Webserver.


Füge das Zertifikat zu
den Signaturgeber CA – Zertifikaten im Java Control Panel unter
„Sicherheit“ → „Zertifikate“.


Navigiere mit deinem
Browser zu ddieser Stelle und starte die launch jnlp-Datei.


Man kann eine Verknüpfung
aus dem Java-Control-Panel erstellen indem man unter „Allgemein“
→ „Ansicht“  mit der rechten Maustaste auf jPrinterAdmin klickt
und „Verknüpfung installieren anklickt.


Tipp: Wenn man die gleiche
JAR-Datei für Webstart und den Server-Prozess verwendet, habe die
beiden immer die gleiche Version.









1.2 Wähle die Datenbank


Wenn du nicht mehrere
Benutzer brauchst und keine Server-Prozess installieren möchtest ist
Lokal (Sqlite) eine gute Wahl. Ansonsten musst du zwischen
MS-SQL-Server und MYSQL wählen. Das ist notwendig wenn mehrere
Benutzer auf die Datenbank zugreifen sollen oder ein Server-Prozess
in regelmäßigen Abständen die Druckerdaten auslesen soll.



1.3 Erstellen der
Datenbank und Aufbau der Verbindung



Klicke im Menü auf „Einstellungen“ und dann auf „Verbindung“.
Das Verbindungsfenster öffnet sich. Klicke auf die Schaltfläche
„Datenbank erstellen“ und wähle im Auswahlfeld die Datenbank.
Falls du lokal wählst musst du nur auf „Datenbank erstellen“
klicken. jprinteradmin erstellt dann eine sqlite-Datenbank-Datei in
deinem Anwendungsdaten-Ordner im Unterordner ".jprinteradmin".
Wenn du eine der anderen wählst musst du Server, Port ( der Standard
ist vorgegeben ), Administrator-Benutzerdaten, der Name der Datenbank
die erstellt wird und Benutzername und Passwort für den neu zu
erstellenden Benutzer, der dann für die Verbindung zur Datenbank
benutzt wird.


Klicke auf „Erstellen“








Tipp: Wenn ein Fehler
aufgetreten ist muss im Falle von einer lokalen Datenbank die
sqlite-Datenbank-Datei gelöscht werden, bei den anderen muss die
Datenbank sowie der erstellte Benutzer gelöscht werden.



1.4 Einstellung der Netzwerk-Umgebung








Klicke im Menü auf
„Einstellungen“ und dann auf „Netzwerk“.


Trage die erste und die
letzte IP_Adresse ein uns gib dem Bereich einen Namen. Dass kann
nützlich sein, wenn man verschieden Netzwerkbereiche in
unterschiedlichen Standorten hat.


Klicke auf die
Schaltfläche „Hinzufügen“.


Beispiel


erste IP-Adresse:
192.168.2.200


letzte IP-Adresse:
192.168.2.250


Kunde: Standard-Bereich



1.5 Teste deine
Konfiguration








Klicke im Hauptfenster auf
„lese Druckerdaten“. Das Programm sollte jetzt jeder IP-Adresse
nach Drucker überprüfen und anschließende diese im Hauptfenster
anzeigen. Durch eine Doppel-Klick auf einen Eintrag erreicht man das
Drucker-Fenster in dem alle Abfrage-Werte des Druckers angezeigt
werden.









2. Wir geht es weiter



2.1 Einstellungen der
Werte die von den Druckern gelesen werden sollen








Gehe zu „Einstellungen“
→ „Einstellung Abfrage-Werte“








Füge ein Feld zu der
Standard-Abfrage Methode hinzu, indem du einen Namen eingibst und die
anderen Felder ausfüllst:


- Sichtbar in Hauptfenster
→ ob dieser Wert im Hauptfenster zu sehen ist


- Reihenfolge → an
welcher Stelle es Sichtbar ist (wenn es Sichtbar ist)


- Such-Feld → Ob der
Inhalt des Feldes in der Suche im Hauptfenster berücksichtigt wird.


- Identifikations-Feld →
ob dieses Feld zur Identifikation des Druckers benutzt wird


- der
Standard hierfür ist die MAC-Adresse, die IP wäre auch möglich


- Zähler → Wenn ein
Feld als Zähler angesehen wird, werden alle Abfragewerte gespeichert
und es ist Möglich Summen- und Durchschnitts-Werte zu errechnen.








Anschließend klicke auf
„Hinzufügen“. Jetzt ist das Feld hinzugefügt. Aber dem Programm
ist immer noch nicht bekannt wie es den Wert ermitteln soll.

Methode in dem Auswahlfeld:

bekannte IP-Adresse eingeben und mit einem Klick auf „Testen“

2. HTML- das ist etwas

4. userdefined – diese
Werte können vom Benutzer im Haupt-Fenster oder dem Drucker-Fenster
manuell vergeben werden.








Wenn du eine andere
Abfragemethode brauchst, normalerweise weil du verschieden Drucker
hat bei dem du den gleichen Wert mit unterschiedlichen Aktionen
abfragen musst, schreibe einen Namen in das Textfeld oben rechts im

Danach kannst du die neue

Das hat nur dann einen
Effekt wenn man die Methode einem Druckertyp zuweist (siehe Punkt

2.2 Druckertypen definieren


Klicke auf
„Einstellungen“ im Menü und anschließend auf „Drucker-Typen“.


Hier kannst du alle bereits gefundenen Druckertype
sehen. Du kannst hier die Abfragemethode für einen Drucker ändern
indem di ihn anklickst und anschließend eine Abfrage-Methode
auswählst ( Die Definition dieser ist in Punkt 2.1 beschrieben ) und
auf „Ändern“ klickst. Man kann hier auch einen neuen Drucker-Typ
eingeben. Normalerweise macht das allerdings keinen Sinn, da man
nicht genau weiß wie der Druckertyp in der SNMP-Antwort heißt bevor
man es getestet hat.











2.3 Bestimmen der allgemeinen Einstellungen


Klicke auf „Einstellungen“ im Menü und
anschließend auf „Allgemeine Einstellungen“.


Hier kann man die allgemeinen Einstellung, die für
alle Benutzer und Server gelten, bestimmen.


Auf der rechten Seite findest du die Einstellungen
für die Drucker-Abfrage und auf der linke die für den
Email-SMTP-Server. Das ist nur wichtig wenn du einen Bericht per
E-Mail verschicken willst.


Beachte, dass der SMTP-Server mit Port eingegeben
werden muss. Zum Beispiel. 192.168.2.1:25 für einen
SMTP-Server ohne SSL-Anmledung.

2.5 Einrichte eines Server-Prozesses


Dieses Beispiel ist für einen Linux-Server, aber es
sollte kein Problem für einen Windows-Administrator darstellen das
Vorgehen zu übertragen um einen  “Windows Scheduled Task”
erstellen.


Java Version 7 oder höher muss installiert sein.


Erstellen einen Benutzer der den Prozess startet,


z.B. “sudo useradd jprinteradmin”.


Kopiere deine “appdata – .jprinteradmin” zu dem
appdata-Ordner des Benutzers auf dem Server z.B. nach
/home/jprinteradmin/.jprinteradmin


Erstelle einen cronjob der einmal am Tag läuft indem
Du eine Zeile zur Datei “/etc/crontab” hinzufügst.


z.B. 



“35  7    * * *   jprinteradmin java -jar
/var/www/jPrinterAdmin/jPrinterAdmin.jar planer”










In diesem Fall wird der Task jeden morgen um 07:35
vom Benutzer “jprinteradmin.” ausgeführt.


Der Prozess wird dann nachsehen, ob etwas zu tun ist.


Um zu definieren wann etwas zu tun ist lies weiter
bei Punkt 2.6.

3 Spezielle Tätigkeiten



3.1 Wechseln zu einer anderen Datenbank


Das kann zum Beispiel Sinnvoll sein, wenn man mit
einer lokalen Datenbank begonnen hat, aber jetzt einen Server-Prozess
einrichten möchte und daher eine Datenbank braucht die immer
verfügbar ist und mehrere Benutzer erlaubt.


Um die Datenbank zu wechseln klicke auf „Datei“ →
„Exportieren“, wähle den Ordner und gib dem Job einen Namen,
anschließen klicke auf „Ausführen“.


Anschließend kannst du einen neue Datenbank
erstellen und jPrinterAdmin mit dieser verbinden. (wie beschrieben in
Punkt 1.2)


Zum Schluss kannst du
die erstellte Datei wieder importieren.
Dazu klicke auf „Datei“ → „Importieren“, wähle die Datei
und klicke auf „Öffnen“.



3.2 Löschen eines Locks


Gehe zu „Extras“ - „Lock betrachten“.


Hier kannst du die ganze Lock-Tabelle löschen oder
einen Einzelnen, indem du ihn anklickst und anschließend auf die
Schaltfläche „Löschen“ klickst.
